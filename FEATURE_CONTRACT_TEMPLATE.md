# Feature Contract Template

**목적**: 새 기능 구현 전에 계약을 먼저 정의한다.  
         이 템플릿을 복사하여 PR description 또는 ADR에 포함시킨다.

**사용 시점**: 새 기능 개발 시작 전. PR 오픈 시 description에 포함.

---

## Feature Contract

### 1. 기능 이름과 목적

```
기능 이름:
목적 (한 문장):
해결하는 문제:
```

### 2. 사용자 영향

```
영향받는 사용자:
UX 변화 (화면/흐름):
에러 발생 시 사용자 경험:
```

### 3. 기술 계약

#### API 계약 (해당 시)
```
엔드포인트:
메서드:
입력:
성공 응답:
에러 응답:
```

#### 상태 계약
```
loading 상태:
empty 상태:
error 상태:
success 상태:
```

#### 권한 계약
```
필요 권한:
인증 불가 시 처리:
```

### 4. 계층 배치

```
비즈니스 로직 위치:
UI 컴포넌트 위치:
API 라우트 위치 (해당 시):
```

### 5. 테스트 계획

```
단위 테스트 대상:
통합 테스트 대상:
E2E 필요 여부:
```

### 6. 완료 기준 (Definition of Done)

- [ ] API 계약을 만족하는 구현 완료
- [ ] loading/empty/error/success 상태 구현
- [ ] check:* 통과
- [ ] TypeScript strict 통과
- [ ] 관련 Constitution 규칙 준수 확인

---

## 사용 방법

1. 이 파일을 복사한다
2. 새 기능 PR의 description에 붙여넣는다
3. 각 항목을 채우고 PR 리뷰어와 함께 검토한다
4. 큰 기능이면 `docs/adr/` 에 ADR로 기록한다
