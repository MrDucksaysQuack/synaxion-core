# Emotional State Patterns

> **Synaxion Constitution 18장**  
> 논리 유형만으로 UI를 결정하지 않는다.  
> 사용자의 감정 상태가 인지 방식에 영향을 미치며, UI는 이를 보조해야 한다.

---

## §0 왜 감정 레이어가 필요한가

동일한 Logic Type이라도 사용자의 **감정 상태**에 따라 필요한 UI가 다르다.

예시:
```
Sequential 논리 + 불안한 사용자 → Progress clarity 필수
Sequential 논리 + 탐색 중인 사용자 → 각 단계의 미리보기 제공
Sequential 논리 + 압박받는 사용자 → 단계 수 최소화, 선택지 단순화
```

감정 레이어는 Logic Type 결정 **이후**, 컴포넌트 선택 **이전**에 적용한다.

---

## §1 5가지 감정 상태

---

### 1. 불안 (Anxiety)

**발생 상황**: 진행 중인 액션, 결과를 기다리는 상태, 처음 접하는 복잡한 시스템

**사용자 내면**:
```
"이게 제대로 되고 있는 건가?"
"얼마나 더 해야 하지?"
"내가 실수한 건 아닐까?"
```

**UI 대응 원칙**: **Progress Clarity** — 지금 어디에 있고, 얼마나 남았는지 항상 보여준다

| 요소 | 구현 |
|------|------|
| 진행 위치 표시 | Step indicator, Breadcrumb, "3/7 단계" |
| 예상 시간 | "약 2분 소요", "3단계 남음" |
| 현재 상태 확인 | 완료된 항목에 체크 표시 |
| 오류 즉각 표시 | 입력 중 실시간 검증 (제출 후가 아님) |
| 저장 상태 | "자동 저장됨" 알림 |

**Truefarm 적용**: Join Wizard — Step indicator + "draft 자동 저장됨"  
**Agrinovation 적용**: 검증 제출 후 — "검토 중입니다. 보통 24시간 내 완료됩니다"

---

### 2. 탐색 (Exploration)

**발생 상황**: 첫 방문, 아직 결정하지 않음, "어떤 것들이 있는지" 보는 중

**사용자 내면**:
```
"어떤 게 있는지 보고 싶다"
"내게 맞는 게 있을까?"
"강요받고 싶지 않다"
```

**UI 대응 원칙**: **Relational Discovery** — 관계와 연결을 자연스럽게 드러낸다. 강제하지 않는다.

| 요소 | 구현 |
|------|------|
| 점진적 정보 공개 | Scroll reveal, hover detail |
| 낮은 commitment CTA | "더 알아보기" (가입 강요 금지) |
| 관계 시각화 | "이것을 보는 사람들이 자주 보는 것" |
| 필터링 | 관심 있는 것만 볼 수 있게 |
| 되돌아가기 쉬움 | 명확한 back/close 경로 |

**Truefarm 적용**: Home 스크롤 — 각 섹션을 강요 없이 탐색  
**Agrinovation 적용**: 프로그램 목록 — 역할 필터 + 카드 미리보기

---

### 3. 확신 필요 (Confirmation)

**발생 상황**: 중요한 결정 직전, 돈/시간/관계가 걸린 상황, 한 번 하면 되돌리기 어려운 선택

**사용자 내면**:
```
"이게 정말 맞는 선택인가?"
"다른 옵션을 놓치고 있는 건 아닌가?"
"이 선택의 결과가 뭔지 알고 싶다"
```

**UI 대응 원칙**: **Comparative + Social Proof** — 비교 정보와 신뢰 신호를 함께 제공한다

| 요소 | 구현 |
|------|------|
| 선택지 비교 | Side-by-side, 장단점 표시 |
| 결과 미리보기 | "이걸 선택하면 → 이렇게 됩니다" |
| 사회적 증거 | "500명의 농가가 이 방법으로 시작했습니다" |
| 취소/변경 가능성 | "나중에 변경 가능합니다" 명시 |
| 위험 최소화 | "먼저 무료로 시작할 수 있습니다" |

**Truefarm 적용**: Join 역할 선택 — 각 역할의 설명 + "이 역할이 맞는지 확인하는 방법"  
**Agrinovation 적용**: Batch 제출 전 — 입력 요약 확인 화면

---

### 4. 신뢰 부족 (Distrust)

**발생 상황**: 처음 접하는 조직, 복잡한 구조, 어디서 왔는지 모르는 시스템

**사용자 내면**:
```
"이 조직은 믿을 수 있는가?"
"내 데이터가 어떻게 쓰이는가?"
"누가 뒤에 있는가?"
```

**UI 대응 원칙**: **Governance Visibility** — 구조와 책임을 명확히 보여준다

| 요소 | 구현 |
|------|------|
| 구조 시각화 | Trust Structure, 계층 다이어그램 |
| 책임 명시 | "누가 이 데이터를 관리하는가" |
| 검증 가능한 증거 | 인증, 파트너십, 실제 사례 |
| 투명한 프로세스 | "어떻게 작동하는가" 공개 |
| 연락 경로 | 쉽게 찾을 수 있는 Contact |
| 개인정보 명시 | Privacy policy 링크 (숨기지 않음) |

**Truefarm 적용**: Trust Structure 섹션 — Agrinovation OS 레이어 시각화  
**Agrinovation 적용**: Auditor 역할 — 검증 이력 공개, Append-only 설명

---

### 5. 압박 (Pressure)

**발생 상황**: 에러 발생, 마감 임박, 다시 해야 하는 상황, 이미 지쳐있는 상태

**사용자 내면**:
```
"빨리 해결하고 싶다"
"뭘 해야 하는지만 알려줘"
"복잡한 설명 필요 없어"
```

**UI 대응 원칙**: **Chunking 강화 + Recovery CTA** — 최소한의 정보와 명확한 다음 행동만 제공한다

| 요소 | 구현 |
|------|------|
| 선택지 최소화 | 1개의 명확한 다음 행동 |
| 에러 설명 단순화 | "무엇이 잘못됐는가" (기술 코드 X) |
| Recovery CTA | "다시 시도", "지원 요청" 버튼 |
| 데이터 보존 | "작성 내용은 저장되었습니다" |
| 탈출 경로 | 저장 후 나중에 돌아올 수 있음을 알림 |

**Truefarm 적용**: 폼 제출 실패 — 에러 메시지 + "다시 시도" + "지원 문의"  
**Agrinovation 적용**: 검증 실패 — 실패 이유 + 수정 방법 + 재제출 CTA

---

## §2 감정 상태 감지 신호

감정 상태는 사용자가 직접 말하지 않는다. 아래 신호로 추론한다:

| 감정 상태 | UI/상황 신호 |
|-----------|-------------|
| 불안 | 폼 진행 중, 결제 단계, 긴 Wizard |
| 탐색 | 첫 방문, Landing page, 카탈로그 |
| 확신 필요 | 최종 확인 페이지, 역할/플랜 선택 직전 |
| 신뢰 부족 | 처음 접하는 About/Trust 페이지 |
| 압박 | Error state, 실패 후 재시도, 마감 컨텍스트 |

## §3 Logic Type × Emotional State 매트릭스

| Logic Type | 불안 | 탐색 | 확신 | 신뢰 부족 | 압박 |
|------------|------|------|------|-----------|------|
| Sequential | Progress indicator 강화 | 각 단계 미리보기 | 결과 요약 화면 | 단계 투명화 | 단계 수 최소화 |
| Conditional | 선택 결과 미리보기 | 역할 설명 상세화 | 비교 옵션 제공 | 각 분기의 의미 명시 | 선택지 2개로 압축 |
| Hierarchical | 현재 위치 표시 | 계층 탐색 자유도 | 내 위치 확인 | 전체 구조 공개 | 내 레이어만 표시 |
| State-based | 상태 변화 실시간 표시 | 상태별 설명 | 다음 상태 미리보기 | 상태 기록 공개 | Recovery CTA만 |
| Comparative | 선택 후 변경 가능 명시 | 필터로 좁히기 | 차이점 강조 | 비교 기준 명시 | 핵심 차이 1개만 |

---

## §4 감정 레이어 적용 체크리스트

```
[ ] 이 화면에서 사용자의 주요 감정 상태는?
[ ] 해당 감정 상태의 UI 대응 원칙이 적용되었는가?
[ ] Logic Type과 감정 레이어가 충돌하지 않는가?
[ ] 불안 상태라면 → Progress Clarity가 있는가?
[ ] 신뢰 부족이라면 → 구조와 책임이 시각화되었는가?
[ ] 압박 상태라면 → CTA가 1개이고 선택지가 단순한가?
```
