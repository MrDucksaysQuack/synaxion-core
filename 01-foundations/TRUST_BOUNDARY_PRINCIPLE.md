# Trust Boundary 원칙 (신뢰 한계)

**목적**: 시스템이 **어디까지 책임지고, 어디서부터 책임을 지지 않는지**를 문서로 선언하는 범용 원칙을 정의합니다.  
**Tier**: 2 (권장. 보안·운영·기대 관리에 유리)

---

## 1. 핵심 원칙

- **명확성**: 시스템의 기술적·서비스적 신뢰 한계를 **문서로 명시**한다.
- **투명성**: 개발자·운영자·(필요 시) 사용자가 한계를 이해할 수 있도록 한다.
- **책임 분리**: "시스템이 책임지는 것"과 "외부 서비스·사용자·제3자가 책임지는 것"을 구분한다.

---

## 2. 적용 방식

- 프로젝트별로 **Trust Boundary 선언 문서**를 두고, 다음을 기술한다.  
  - **시스템이 책임지는 것**: 예) 데이터 저장·검증·표시, 내부 API 가용성, 권한 체크.  
  - **시스템이 책임지지 않는 것**: 예) 외부 API 정확도·가용성, 사용자 입력 내용의 진실성, 제3자 서비스 장애.
- 도메인별(예: OCR, 결제, 인증) 세부 항목은 프로젝트 인스턴스 문서에서 정의.
- 이 원칙은 "한 문서를 두고 유지·갱신한다"는 **의무**를 의미하며, 구체적 문구는 프로젝트에 맡긴다.
