## 충돌·누락 유형 체크리스트

```
충돌 체크:
[ ] 문서 role명 ≠ DB role명
[ ] API 기록 필드 ≠ 다른 서비스 읽기 필드
[ ] UI 상태값 ≠ DB enum 값
[ ] test mock 응답 ≠ 실제 API 응답 구조
[ ] env.example ≠ 실제 사용 env var

누락 체크 (Synaxion Safety):
[ ] UI에 있는 기능의 API 없음
[ ] DB table에 대한 RLS 없음
[ ] 외부 서비스 호출에 타임아웃 없음        ← 04-safety-standards
[ ] 에러 처리 없는 async 함수              ← 04-safety-standards
[ ] 침묵 실패 패턴 (에러를 삼키는 코드)     ← 04-safety-standards
[ ] 타입 검증 없는 외부 입력
[ ] 테스트 없는 핵심 비즈니스 로직
```
